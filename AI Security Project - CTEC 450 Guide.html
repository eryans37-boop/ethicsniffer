1 Check Your Python Version

Open a terminal (Command Prompt, PowerShell, or Terminal on Mac/Linux) and run:

python --version

You need Python 3.9 or higher. If you see an error or an old version, go to python.org and download the latest version before continuing.

2 Install Required Packages

Install PyTorch (CPU-only version is fine and much faster to download) and matplotlib:

# Install PyTorch — CPU only version (smaller download, works fine for MNIST)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu

# Install matplotlib for saving charts and images
pip install matplotlib

This will take a few minutes. When it finishes, verify it worked:

python -c "import torch; print(torch.__version__)"

You should see a version number (e.g., 2.3.0+cpu). If you get an error, re-run the install command.

Windows note: If pip is not recognized, try python -m pip install ... instead.

3 Run the Code — Train the Baseline Model

Navigate to the project folder and run the main script:

python "AI Security Project - CTEC 450.py"

The script will automatically:

  1. Download the MNIST dataset (about 11 MB) the first time you run it
  2. Train a convolutional neural network for 5 epochs
  3. Print accuracy after each epoch
  4. Run the FGSM attack and print the accuracy under attack
  5. Run adversarial training for 3 more epochs
  6. Print final clean and attack accuracy for the defended model
  7. Save results.png and fgsm_comparison.png to the current folder
  8. Save baseline_model.pth and defended_model.pth (the trained weights)

5 Write Down the Exact Numbers

Before doing anything else, copy the accuracy numbers from your terminal output and write them down. You will use these in the paper and the slides. Never change or estimate these numbers — put the exact values your code produced.

Your numbers should be close to these. Small differences (±0.5%) are normal because of random weight initialization:

ConditionAccuracy

Under FGSM attack (ε=0.3)~96–97%
Defended, clean images~99–99.3%

If your attack only drops accuracy by 2–3%, that is correct and expected. MNIST is a simple dataset. The attack would be more destructive on harder data like CIFAR-10.

6 How the Code Works — What to Know for the Paper

The Neural Network Architecture

Input: 28x28 grayscale image (1 channel)
  → Conv2d(1, 32, kernel=3x3) → ReLU → MaxPool(2x2)
  → Conv2d(32, 64, kernel=3x3) → ReLU → MaxPool(2x2)
  → Flatten
  → Linear(64 × 7 × 7, 128) → ReLU
  → Linear(128, 10)
Output: 10 scores (one per digit 0–9)

Training Details

SettingValue
OptimizerAdam (learning rate = 0.001)
Loss functionCross-entropy
Epochs5 baseline + 3 adversarial
Batch size64
Training samples60,000
Test samples10,000

FGSM Attack Formula

x_adv = x + ε · sign(∇_x J(θ, x, y))

Where x is the original image, ε (epsilon) controls the strength of the perturbation, and the gradient shows which direction to push each pixel to increase the model's error.

Adversarial Training Defense

The defense loads the saved baseline model and trains it for 3 more epochs, but this time it generates FGSM adversarial examples during training and trains on those instead of clean images. This teaches the model to handle adversarial inputs correctly. The training uses ε=0.2 (slightly weaker than the test attack at ε=0.3) which is intentional — exposing the model to slightly weaker attacks during training makes it generalize better.
