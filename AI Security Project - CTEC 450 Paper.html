Abstract Machine learning models are becoming more common in security-sensitive applications, but they have a weakness that is not obvious at first glance: they can be tricked by inputs that have been modified in a specific way that is invisible to the human eye. This paper documents an experiment where I trained a convolutional neural network (CNN) on the MNIST handwritten digit dataset, attacked it using the Fast Gradient Sign Method (FGSM), and then applied adversarial training as a defense. The baseline model reached 99.01% accuracy on clean images. After the FGSM attack with an epsilon of 0.3, accuracy dropped to 96.42% — a drop of 2.59 percentage points. After adversarial training, accuracy under the same attack recovered to 98.25%, while clean accuracy improved slightly to 99.30%. The results show that MNIST's relative simplicity limits how much a single-step attack can degrade a robust model, but the effect is real and measurable, and adversarial training demonstrably strengthens robustness against it.

Introduction

I went into this project not really understanding what an adversarial attack on a machine learning model actually meant in practice. I had heard the term before — something about images that fool neural networks — but I did not have a solid mental model of how it worked or how bad the impact could be.

The answer turned out to be more nuanced than I expected. A mathematically calculated noise pattern added to an image can measurably drop model accuracy — in my case from 99.01% down to 96.42% under attack. That might sound like a small drop, but on a dataset as clean and simple as MNIST it is meaningful, and the same attack on a harder dataset or a model deployed in production would be significantly worse.

This matters because machine learning models are increasingly used in real applications: facial recognition at airports, fraud detection in banking, malware classifiers in antivirus software. Understanding how these attacks work, and how to defend against them, is a core skill in modern cybersecurity.

This paper covers three things: how I built the baseline model, how I implemented the FGSM attack, and how adversarial training improved robustness. All code was written in Python using PyTorch and the MNIST dataset (LeCun et al., 1998).

Description of the Attack: Fast Gradient Sign Method

FGSM was introduced by Goodfellow et al. (2015) in “Explaining and Harnessing Adversarial Examples.” The core idea is simple: instead of computing gradients with respect to the model's weights (as in training), compute them with respect to the input image, then nudge each pixel in the direction that increases loss. The formula is:

xadv  =  x  +  ε · sign( ∇x J(θ, x, y) )

Where x is the original image, ε is attack strength, ∇x J is the gradient of loss with respect to the input, and sign() extracts only the direction of that gradient.

The result is an image that looks identical to the original but the model classifies it differently. This experiment used ε = 0.3.

Methodology

Environment

  • Python 3.12  |  PyTorch 2.3  |  torchvision 0.18
  • matplotlib 3.9  |  numpy 2.4
  • CPU (no GPU required — MNIST is small enough to train in minutes)

Step 1 — Baseline Model

A simple CNN: two convolutional layers followed by two fully connected layers.

Conv2d(1, 32, 3×3)  → ReLU → MaxPool(2×2)
Conv2d(32, 64, 3×3) → ReLU → MaxPool(2×2)
Linear(64×7×7, 128) → ReLU
Linear(128, 10)

Trained for 5 epochs with Adam (lr = 0.001) and cross-entropy loss on 60,000 MNIST training images, evaluated on 10,000 test images (LeCun et al., 1998).

Step 2 — FGSM Attack

Implemented from scratch following Goodfellow et al. (2015). For each test batch: forward pass to compute loss, backward pass with images.requires_grad = True to get input gradients, apply the signed perturbation, re-evaluate on the perturbed images. No model weights changed during the attack.

Step 3 — Adversarial Training Defense

Starting from the saved baseline weights, the model was retrained for 3 additional epochs where each training batch was converted to FGSM adversarial examples (ε = 0.2). A halved learning rate (0.0005) was used to avoid disrupting the already well-trained weights.

What the Numbers Mean

The drop from 99.01% to 96.42% is 2.59 percentage points. That is smaller than what is typically shown in adversarial ML papers, and there is a clear reason. MNIST is an unusually easy dataset — the digit shapes are simple and distinct, and a well-trained CNN learns very confident, stable features. A single-step FGSM attack is not enough to fully break those features. On a harder dataset like CIFAR-10, where the model relies on subtler texture patterns, the same attack would cause much larger accuracy drops.

Even so, the effect is real. A 2.59% drop on 10,000 test images means roughly 259 images that the model was correctly classifying are now misclassified — entirely because of pixel changes invisible to the human eye. That is a real vulnerability, not a rounding error.

The defense improved performance across the board. The defended model hit 99.30% on clean images (slightly better than baseline) and 98.25% under attack, recovering 1.83 percentage points compared to the undefended model under attack. There was no clean-accuracy trade-off, which is partly because MNIST is simple enough that adversarial training examples reinforced rather than conflicted with clean-data patterns (Madry et al., 2018).

Defense Approach

The defense implemented is adversarial training (Goodfellow et al., 2015; Madry et al., 2018). By including FGSM-generated adversarial examples during training, the model learns to classify them correctly rather than only seeing clean data.

Madry et al. (2018) showed that PGD-based adversarial training — which uses multiple gradient steps to find stronger adversarial examples — produces more robust models than single-step FGSM training. This project used FGSM for training because it is faster to compute and still demonstrated a clear improvement. PGD training would be the logical next step, especially on harder datasets where the single-step attack is more damaging.

Other defenses considered but not implemented:

  • Input preprocessing — smoothing or median filtering to reduce perturbation effects; can be defeated by adaptive attacks
  • Detection-only — flag adversarial inputs without classifying them; limits damage but does not restore accuracy
  • Ensemble models — require multiple models to agree; more robust but computationally expensive

Conclusion

This project made adversarial attacks concrete in a way that reading about them does not. Taking a real MNIST test image, adding an invisible perturbation, and watching the model misclassify it — that is a real demonstration of a real security issue, even if MNIST keeps the numbers modest.

The verified results: baseline 99.01%, under FGSM 96.42%, defended under attack 98.25%. The defense worked cleanly with no accuracy trade-off on clean data. The broader lesson is that the attack technique is dataset-independent — it works the same way on facial recognition or malware classifiers, where the accuracy drops would be far more severe. High clean accuracy does not mean a model is secure.
