What This Project Is About

Machine learning models can be fooled by tiny, invisible changes to their input data. This project proves that claim with real code and real numbers.

  • Train a CNN classifier on MNIST handwritten digits
  • Attack it using FGSM — a mathematically designed adversarial perturbation
  • Defend it using adversarial training
  • Measure the accuracy at each stage
Why it matters: AI models are used in facial recognition, malware detection, fraud prevention. If an attacker can fool the model, the whole system fails — even if clean accuracy is near 100%.

The Model: SimpleCNN on MNIST

Architecture

Conv2d(1→32, 3×3) + ReLU + MaxPool
Conv2d(32→64, 3×3) + ReLU + MaxPool
Linear(64×7×7 → 128) + ReLU
Linear(128 → 10 classes)

Training Setup

  • Dataset: MNIST (60K train / 10K test)
  • Optimizer: Adam (lr = 0.001)
  • Loss: Cross-entropy
  • Epochs: 5  |  Batch size: 64

Why This Architecture?

Two conv layers give the model enough capacity to hit ~99% accuracy on MNIST without being so complex that it is hard to train or explain. The architecture is intentionally simple — this is a learning project, not a production system.

The Attack: Fast Gradient Sign Method

Introduced by Goodfellow et al. (2015). The idea: instead of changing model weights to reduce loss, change the input pixels to increase loss.

x_adv = x + ε · sign( ∇_x J(θ, x, y) )
  • x — original image pixel values
  • ε (epsilon) — how strong the attack is (0.3 in this project)
  • ∇_x J — gradient of loss with respect to the input image
  • sign(...) — only the direction matters, not the size
The perturbation is invisible to a person but devastating to the model. FGSM is a single-step attack — it is fast and easy to implement, which is why it is used as a standard benchmark for adversarial robustness.

What to Notice

  • Some digits are misclassified despite being visually unchanged
  • There is a faint static pattern visible if you look closely — that is the adversarial noise
This is the key point: the attack is imperceptible to humans but effective against the model.

Defense: Adversarial Training

Train the model on adversarial examples so it learns to classify them correctly, not just clean images.

How It Works

for each training batch:
    1. generate FGSM adversarial examples (ε = 0.2)
    2. train the model on the adversarial batch
    3. repeat for 3 epochs

Key Choices

  • Start from the saved baseline weights
  • Use ε = 0.2 for training (slightly weaker than the ε = 0.3 test attack)
  • Lower learning rate (0.0005) to avoid catastrophic forgetting
  • 3 extra epochs — enough to improve without overfitting

Why This Defense?

  • Directly addresses the vulnerability
  • Results are measurable and reproducible
  • Simple to implement in PyTorch
  • Well-studied in the literature (Madry et al., 2018)

Conclusion & Lessons Learned

What I Proved

  • Verified: 99.01% baseline → 96.42% under FGSM (real MNIST run)
  • Adversarial training recovered to 98.25% under the same attack
  • No clean-accuracy trade-off — defense improved both metrics
  • Smaller drop than typical because MNIST is simple — same attack hits harder on real datasets

Next Steps

  • Try PGD attack (stronger, multi-step)
  • Compare two defenses side by side
  • Test on CIFAR-10 (harder dataset)
The big takeaway:

High clean accuracy does not mean a model is secure. If you are deploying ML in a security-critical system, you have to think about adversarial robustness from the start — not as an afterthought.
